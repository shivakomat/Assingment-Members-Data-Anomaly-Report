Assignment Solution Overview

The application solves it by creating a stream of data and traversing row by row where it goes through the process of validating the columns data and then outputting the invalid member’s records and deleted records to two seperate files.

One file representing the invalid members based on the validations check and another file representing the deleted records.

Please see the application settings section and how to run the app section to get your results and further even testing against other members data sets.

The file outputs structure explanation below:

Invalid records file has four extra columns on top of the rest of the members columns to show validations status. Each column represents a boolean value, false represents invalid for the respective column’s data and vice versa.

Further you can use the flat result files in any way to filter the data. For example the files can also be used by dumping into a database for easier analysis using SQL instead of Excel.

Deleted records csv file contains just one column which represents the id’s of the member’s that were deleted or in other words they are missing from the source data set.

Assumptions

  • The data sets are pre sorted
  • The date formats are the same as the data the set I received for all the data columns, this is because the format is embedded in code.
  • The data format will always be JSON

Application Settings

The application code is packaged into a jar file which is executable by scala version 2.12. The jar file also includes all the dependency libraries needed to execute the app. So that leaves us with only one dependency to be installed to run the jar which is Scala version 2.12. Below FAQ will help you install Scala and be able to run the app.

FAQ

How to install scala ?

Assuming that you already have brew installed, follow the below steps:

  1.    brew install scala@2.12 (try scala -version after)
  2.    export PATH=”/usr/local/opt/scala@2.12/bin:$PATH” (optional)
